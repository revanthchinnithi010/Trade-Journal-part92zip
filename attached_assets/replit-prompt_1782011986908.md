# Build: Crypto Perpetuals Trading Screen (Mobile-first)

Build a mobile-first React web app that replicates a crypto perpetual-futures trading screen (BTCUSD), matching a dark-themed trading app like Delta Exchange/Bybit. Three view-states live in one screen, toggled by a chevron button: **Chart + Order Entry**, **Order Book + Order Entry**, and **Contract Details**.

## Tech stack

- React (Vite)
- Tailwind CSS ‚Äî **use only core/default Tailwind utility classes and the default color palette** (gray, emerald, rose, orange). No arbitrary `bg-[#hex]` values, no custom config colors.
- `lucide-react` for icons
- No backend ‚Äî all data is local mock/derived state (deterministic pseudo-random, not `Math.random()`, so the UI doesn't flicker on re-render)

## Global layout

- Root container: `bg-black text-gray-100 min-h-screen max-w-sm mx-auto flex flex-col` (phone-width column, centered on wider viewports)
- Sections are **flat, full-bleed strips separated by 1px hairlines** (`border-gray-800`) ‚Äî no card shadows, no rounded section containers. Rounded corners (`rounded-lg`/`rounded-md`) are reserved for buttons, inputs, and badges only.
- Page horizontal padding: `px-4`. Section vertical padding: `py-3` (rows ~`h-11` to `h-14`).
- Bottom nav is pinned to the bottom via `mt-auto` on its wrapper.

## Component-by-component spec

### `AccountModeTabs`
Row: "Main" (gray-400) and "Isolated" (orange-500, bold, underlined with `border-b-2 border-orange-500`), chevron-down pushed to the right with `ml-auto`.

### `SymbolHeader`
Props: `price`, `changePct`, `direction` ("up"|"down").
Left: filled orange star icon + "BTCUSD" (`text-lg font-semibold`) + chevron-down, with "Bitcoin Perpetual" subtitle below.
Right, right-aligned: big price in emerald (up) or rose (down) with a chevron-up/down inline, and the % change below in emerald-500.

### `MarketStatsBar`
Props: `expanded` (bool), `onToggleExpand`.
Left: "24h Vol." and "OI" stat pairs (label `text-xs text-gray-400`, value `text-sm text-gray-100`).
Right: an icon button (32px square, bordered, turns orange/active border when `expanded`) that toggles the view ‚Äî chevron flips up/down; a bell icon button with a small "Price Alert" caption underneath; and (only when **not** expanded) an outline "Chart" button with a sliders icon, `border-orange-500 text-orange-500`.

### View A ‚Äî Chart + Order Entry (default, `expanded = false`)

**`TimeframeTabs`**: horizontal scrollable row of pills `5s 1m 5m 15m 1h 4h`. Active pill: `bg-gray-800 text-gray-100 font-medium`, inactive: `text-gray-500`. Local `useState` for active frame.

**`CandlestickChart`**: SVG-based candlestick chart (no external chart library required, but Recharts is fine as an alternative). Generate ~70 candles from a seeded pseudo-random walk starting at price 64000. Up candles `#10B981`/emerald, down candles `#F43F5E`/rose. Draw wick as a thin vertical line and body as a rect. Render a small volume histogram beneath the price chart using the same up/down coloring at reduced opacity. Add 4‚Äì5 horizontal gridlines (`stroke-gray-800`). Add a 3-label time axis below (e.g. `15:00 18:00 21:00`).

**`FundingRow`**: label "Funding (8h) / Countdown" (`text-xs text-gray-400`) with value "0.0100% / 06h:26m:04s" below (`text-sm text-gray-100`).

Below this, render the **Order Book + Order Entry split** (see View B) so chart and trading panel are visible together, exactly like the reference screens.

### View B ‚Äî Order Book + Order Entry split

Container: `flex` row. Order book takes the remaining width (`flex-1`), order entry panel is fixed-ish at `w-[44%] min-w-[160px]` with a left divider `border-l border-gray-800`.

**`OrderBookPanel`**
- Header row: "Price (USD)" / "Size (BTC)" in `text-xs text-gray-500`.
- 8 ask rows (reverse order, highest at top, nearest the spread at bottom), each rendered by `OrderBookRow` with `side="ask"`.
- Spread row: large bold current price, rose or emerald depending on last tick direction.
- Two small rows for Index (`I`) and Mark (`M`) prices ‚Äî small bordered square badge with the letter, then the value, both muted gray.
- 8 bid rows below, `side="bid"`.

**`OrderBookRow`** props: `price`, `size`, `maxSize`, `side`.
- `relative h-7` row.
- An absolutely positioned depth bar (`absolute inset-y-0 right-0`) whose width is `(size / maxSize) * 100%`, colored `bg-rose-500/15` for asks and `bg-emerald-500/15` for bids ‚Äî this is the volume-depth visualization seen in the reference.
- Price text on top, colored rose (ask) or emerald (bid); size text right-aligned, `text-gray-200`.
- Generate ask/bid mock arrays with a seeded pseudo-random helper, not `Math.random()`.

**`OrderEntryPanel`** ‚Äî vertical stack, `gap-3`:
1. `SideToggle` ‚Äî a two-segment control labeled "Buy | Long" and "Sell | Short". Give it a **slanted/parallelogram look** using `skew-x-[-10deg]` on each half (and `skew-x-[10deg]` on the inner text span to counter-skew the label so it reads upright) with a small negative margin so the halves overlap into a chevron seam. Selected side is a solid fill (`bg-emerald-500`/`bg-rose-500`, white text); unselected side is `bg-gray-900` with colored text only. `useState` for selected side.
2. `SelectField` (reusable, props `label?`, `value`) ‚Äî bordered row, `rounded-lg border border-gray-700 h-11`, value text in orange, trailing chevron-down. Used for "Leverage: 1x" and "Market".
3. `QuantityInput` ‚Äî "Quantity" caption above a bordered/filled row showing "1 Lot = 0.001 BTC" on the left and a "Lot ‚åÑ" chip on the right.
4. `PercentQuickSelect` ‚Äî a thin row of plain-text percentage labels: `10% 25% 50% 75% 100%`, evenly spaced, `text-xs text-gray-500` (display-only, no need to wire to quantity logic unless you want to).
5. `BracketOrderRow` ‚Äî "Bracket Order" label left, outline orange "+ Add TP/SL" button right.
6. `SummaryRow` √ó2 ‚Äî "Funds req." ‚Üí "0.00 USD", "Available Margin" ‚Üí "0 USD". Label muted left, value `text-gray-100` right.
7. Full-width submit button, `h-12 rounded-lg font-bold text-base`, fill color matches the selected side (`bg-emerald-500` for Buy, `bg-rose-500` for Sell), label text = "Buy" or "Sell" matching selection.
8. "Reduce Only" checkbox row (plain `text-sm text-gray-400` with a small bordered square checkbox).
9. `ScalperBadge` ‚Äî bordered pill row: a small bordered square + "Scalper Active" in emerald on the left, and a rounded countdown chip "30m" (`bg-emerald-500/15 text-emerald-400`) on the right.
10. Two side-by-side outline buttons: "Fees" and "Calculator" (`flex-1 h-9 border-gray-700 text-gray-300`).

Below the split, render **`PositionTabs`**: three tab labels ‚Äî "BTCUSD", "Position", "Open Orders (0)" ‚Äî underline style active indicator (`border-b-2 border-orange-500`, bold white) vs inactive (`text-gray-500`, transparent border). `useState` for active tab.

### View C ‚Äî Contract Details (`expanded = true`)

**`ContractStatsGrid`**: 4-column grid (`grid grid-cols-4 gap-2`), each cell has a muted `text-[11px]` label stacked above a `text-sm text-gray-100` value: "Funding" / "0.0100% /8h", "Next Funding In" / "06h:25m:36s", "24h High" / "$64373.5", "24h Low" / "$62919.0".

Below a divider, a `text-base font-bold` "Contract Details" header, then a flat list of `DetailRow` items (each `h-14 flex justify-between items-center border-b border-gray-800`, label muted left, value right):
- Type ‚Üí Linear
- Lot Size ‚Üí 0.001 BTC
- Effective Settlement Currency ‚Üí INR
- Initial Margin ‚Üí 0.5 %
- Max Leverage ‚Üí 200x
- Maintenance Margin ‚Üí 0.25 %
- Underlying Index ‚Üí .DEXBTUSD (orange text + small external-link icon)
- Position Limit ‚Üí 125 BTC, 125,000 contracts
- Status ‚Üí Operational

End with a centered text button "See full contract specifications" + external-link icon, `text-orange-500 text-sm`, full-width row.

### `BottomNav`
Fixed-at-bottom row, 5 evenly-spaced items: Home, Markets, Trade, Charts, Portfolio (icons: `Home`, `TrendingUp`, `ArrowLeftRight`, `Activity`, `Briefcase` from lucide-react). Active item ("Trade") gets an orange filled circle behind a white icon plus orange label text; inactive items are gray icon + gray `text-[10px]` label.

## Behavior / state

- `App.jsx` holds `expanded` state (boolean) controlling whether `MarketStatsBar`'s toggle shows the Chart view or the Contract Details view ‚Äî this is the single source of truth, passed down as a prop + callback.
- `SideToggle` and `PositionTabs` manage their own local `useState`.
- All chart/order-book data should be generated once via `useMemo` with a deterministic seeded-random function so values don't jump on every re-render.

## Responsive behavior

- Mobile-first: the whole app is constrained to `max-w-sm` and centered on larger screens (`mx-auto`), simulating a phone viewport ‚Äî don't build separate desktop/tablet layouts.
- The order book + order entry split stays side-by-side at all widths (this is intentionally a dense trading-app layout, not a stacked consumer layout).
- The candlestick chart SVG should use `preserveAspectRatio="none"` and a `w-full` wrapper so it scales to container width without distortion of the layout grid.
