Make the Long Position and Short Position drawing tool visually identical to TradingView.

IMPORTANT:
Do NOT redesign the tool.
Match TradingView behavior, rendering, opacity, layering, candle interaction, and visual hierarchy exactly.

GOAL:
When a user adds a Long/Short position tool:
- The profit area should dynamically react to candle movement.
- From entry/open price to TP hit candles, show a stronger/thicker active color region.
- Remaining unfilled future area should stay dim and low opacity.
- Same behavior as TradingView Risk/Reward tool.

VISUAL REQUIREMENTS:

1. ACTIVE FILLED AREA
- Detect candles between entry price and TP touch.
- If candles move toward TP:
  - Render a stronger/high-opacity filled region behind those candles.
  - Use a thicker richer color overlay.
  - Example:
    Profit zone:
      activeFillOpacity: 0.32
      activeFillColor: rgba(0, 160, 120, 0.32)

    Loss zone:
      activeFillOpacity: 0.28
      activeFillColor: rgba(180, 30, 30, 0.28)

2. UNFILLED FUTURE AREA
- Remaining area not yet reached by candles should stay dim.
- Use lower opacity like TradingView.

Example:
futureProfitOpacity: 0.10
futureLossOpacity: 0.10

3. DYNAMIC CANDLE CALCULATION
- Continuously calculate:
  entry candle
  current visible candles
  TP touched candle
  SL touched candle

- Stop active fill exactly where TP/SL gets hit.
- Before TP hit:
  active area follows candles only.

4. VERTICAL SPLIT EFFECT
TradingView uses a visual split:
- Left side = active candle-filled region
- Right side = dim future projection region

Implement same split rendering.

5. ENTRY LINE
- Use dashed horizontal entry line.
- Thin soft white line:
  rgba(255,255,255,0.45)

6. LABELS
Match TradingView labels exactly:
- Target label
- Stop label
- P&L label
- RR ratio
- Qty
- Amount

Style:
- Rounded corners
- Backdrop blur
- Semi-transparent background
- Premium typography
- Compact spacing

7. HANDLE DESIGN
Match TradingView handles:
- Blue rounded square
- Outer glow
- Thin border
- Touch friendly

8. SMOOTH RENDERING
- No hard edges
- Anti-aliased rendering
- GPU accelerated transforms
- No jagged opacity transitions

9. LAYERING
Correct z-index order:
candles
→ active filled region
→ dim projection region
→ labels
→ handles

10. INTERACTION
- Real-time update while dragging.
- Smooth resize animation.
- Mobile touch support.
- No flickering.
- Maintain 60fps.

11. EXACT TRADINGVIEW FEEL
The final result must visually match TradingView’s:
- opacity behavior
- candle overlap effect
- active profit highlight
- dim future projection
- label positioning
- interaction smoothness
- premium professional appearance

TECH IMPLEMENTATION:
- Use canvas-based rendering.
- Calculate active candle range using candle timestamps.
- Clip active overlay to candle progress region.
- Use separate alpha layers for active vs inactive sections.
- Use interpolation for smooth transitions.

FINAL TARGET:
The long/short position tool should look almost indistinguishable from TradingView’s native Risk/Reward drawing tool.
